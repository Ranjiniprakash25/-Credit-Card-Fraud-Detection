## 🔍 Problem Statement
Fraudulent transactions are rare (less than 1%) but extremely 
costly. Traditional rule-based systems miss new fraud patterns. 
This model learns patterns automatically from data.

## ⚙️ How It Works
1. Load and explore transaction data
2. Clean and preprocess features
3. Apply feature engineering
4. Train ML classification model
5. Evaluate using precision, recall, F1-score
6. Optimize to reduce false positives

## 📈 Key Learnings
- In fraud detection, precision matters more than accuracy
- Feature engineering improved model performance significantly
- Clean data beats complex algorithms every time
